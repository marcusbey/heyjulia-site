## The 5-stage pipeline

```
[1 Research] → [2 Style Guide] → [3 Synthesis] → [4 QA] → [5 Ship]
```

Each stage writes one artifact into **this** folder:

| Stage | Artifact | Notes |
|-------|----------|-------|
| 1 Research | `snapshot.json` | Scraped facts from existing site, Instagram, Facebook, Google Maps. **Sole** source of truth for all site content. |
| 2 Style Guide | `style_guide.md` | Per-business design system. Never reused across leads. |
| 3 Synthesis | `site/` | Static build (HTML + CSS + vanilla JS, GSAP + ScrollTrigger), or Next.js if CMS/routing needed. Includes `index.html`, `404.html`, `assets/`, `vercel.json`. |
| 4 QA | `qa_report.md` | Anti-hallucination + fidelity checklist (Playbook XIV). Must pass before ship. |
| 5 Ship | `deploy.sh`, `deploy_url.txt`, `deploy_result.json` | Vercel deploy; URL echoed for factory. |

Reference layout: `../../website-generation-skill/output_example/barcola-bistro/`.

## Hard rules (non-negotiable)

From `NEVER_FAKE.md` and `SKILL.md`:

- **Every fact traces to `snapshot.json`.** If data was not scraped, the section is omitted — never fabricated. No lorem ipsum, no stock testimonials, no invented menu items/hours/prices/addresses.
- **Never pure `#000000` or `#ffffff`** — shift toward the temperature chosen in `style_guide.md` (Playbook VII).
- **Never Google Fonts defaults** (Inter / Roboto / Poppins). Paid or distinctive font only (Playbook VI.3).
- **Max 2 fonts, 4 colors, 3 easing curves** (Playbook IX + XIV).
- **At least one "detail nobody asked for"** (Playbook IX.7).
- **Respect `prefers-reduced-motion`.**
- **Full Playbook XIV fidelity checklist must pass** before running `deploy.sh`.

## Commands

The project folder itself has no build system until stage 3 runs. Expect these from inside `site/` once generated:

```bash
# Static build (default): open index.html or serve locally
python3 -m http.server 8000          # preview static site

# Deploy (stage 5) — from this project root, not from site/
./deploy.sh                          # preview deploy
./deploy.sh --prod                   # production
./deploy.sh --project <slug>         # override Vercel project slug
```

Requirements: `vercel` CLI installed globally, `vercel login` completed. `deploy.sh` auto-sources `BASE32 STUDIO PIPELINE/.env` when present. Output is written to `deploy_url.txt` + `deploy_result.json` for the factory scheduler.

If synthesis picks Next.js instead of static, `package.json` will be inside `site/` — use `npm run dev` / `npm run build` from there.
