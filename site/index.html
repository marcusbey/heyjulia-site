01 — How it works

Eliminate jobsite surprises.

Follow along below to see how simple it is to create a project plan and monitor progress in real time.

  1. STEP 1

    Define KPIs & Assign Crew

    Start by listing the key performance indicators for the project, such as tasks or materials. Arrange them by expected completion order and assign crew members to ensure clarity from day one.

    • Easily add tasks with quantities, units, and time estimates.
    • Use AI paste to import task lists from existing documents.
    • Attach drawings or spec sheets directly to your plan items.

  2. STEP 2

    Build Your Schedule

    Transform your list of tasks into a visual project timeline. Drag and drop items and establish clear milestones with a bird's-eye view of progress.

    • Timeline updates dynamically with each completed call.
    • Over time, Julia learns expected completion dates for each task.
    • Adjust crew size to see the new projected finish date.

  3. STEP 3

    Set Up Calls

    Define when and how often to contact your project leaders. Select call days and time windows for automated check-ins and early issue detection.

    • Choose preferred contact days and time windows for calls.
    • Set escalation contacts for urgent issues automatically.
    • Ad hoc calls can be initiated anytime from the office or field.

  4. STEP 4

    Report

    Julia calls the team daily to get progress updates. She speaks naturally with site supervisors to collect progress data, issues, and photos.

    • Julia initiates calls at the scheduled times.
    • Speaks naturally with site supervisors.
    • Collects progress data, issues, and photos.

  5. STEP 5

    Monitor

    Real time dashboards show progress and identify issues before they spiral. Julia translates all updates into a simple red/yellow/green indicator so you can see at a glance which projects need attention.

    • View real-time progress against the schedule and estimate.
    • Identify delays and issues immediately.
    • Julia highlights delays, slowdowns and overruns.
